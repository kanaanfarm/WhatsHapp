CONNECTCHAT PRO - INTERNET DEPLOYMENT CHECKLIST
================================================

This release includes one-to-one audio/video calling. To use it between UAE,
Lebanon, and other countries, deploy one server on the public Internet.

REQUIRED
--------
1. A Linux server/container running Node.js 22 LTS.
2. A domain name, for example chat.yourcompany.com.
3. HTTPS using a reverse proxy such as Caddy or Nginx.
4. WebSocket forwarding for Socket.IO.
5. A TURN service for reliable audio/video calls.
6. A Supabase project for persistent messages and uploaded files.
7. The security-migration.sql script applied in Supabase.

ENVIRONMENT
-----------
Set NODE_ENV=production, PUBLIC_ORIGIN, a stable random SESSION_SECRET of at
least 32 characters, the Supabase variables, and the TURN variables shown in
.env.example. Set CALLS_ENABLED=false when live calls should be disabled. Do
not commit real secrets.

IMPORTANT
---------
- Camera/microphone access works on localhost for testing, but public use must
  be served over HTTPS.
- STUN-only calls are not reliable across all mobile operators and firewalls.
- This package is suitable for controlled pilot/small-team use on one server.
- Supabase-backed sessions and abuse limits are included in version 1.2.0.
- Before commercial or large-scale use, add email/phone verification,
  moderation, malware scanning, monitoring, backups, and a professional review.

WINDOWS EXE AND ANDROID APK
---------------------------
The clients/desktop and clients/android folders contain the installer projects.
The .github/workflows folder contains automatic builders. After the public HTTPS
URL exists, run each workflow and enter that URL. The resulting EXE/APK appears
as a downloadable build artifact. The Android workflow creates a test/debug APK;
Google Play distribution requires a private signing key and a signed release.
